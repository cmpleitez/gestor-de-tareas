El problema: 
php_upstream.conf
conf
upstream php_upstream {
    server 127.0.0.1:9003 weight=1 max_fails=1 fail_timeout=1;
    server 127.0.0.1:9004 weight=1 max_fails=1 fail_timeout=1;
}
Solo tienes 2 workers PHP (puertos 9003 y 9004) con max_fails=1 y fail_timeout=1. Esto significa:

Si un worker falla una sola vez, Nginx lo marca como caído por 1 segundo
Con solo 2 workers, 3 usuarios haciendo peticiones concurrentes pueden saturarlos fácilmente
Cuando ambos están ocupados procesando las peticiones Livewire 
update
 de los otros usuarios, la tercera petición recibe 502 Bad Gateway
💡 Solución recomendada
Aumentar el número de procesos PHP upstream. Según el comentario del propio archivo, se configura en usr/laragon.ini:

ini
[nginx]
Upstream=5
Esto sube a 5 workers PHP, lo que daría margen suficiente para 3+ usuarios concurrentes. Para aplicar el cambio necesitarías reiniciar Laragon.
